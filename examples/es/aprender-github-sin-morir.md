# Aprender GitHub sin morir

## Situación
"Me pierdo con ramas, commits y pull requests".

## Enfoque que abruma
Aprender todo GitHub en un día con 30 conceptos.

## Enfoque centrado en humanos
"Enséñame GitHub para una sola tarea: subir un cambio pequeño y abrir un PR. Explícalo en pasos cortos y dime qué revisar en cada paso".

## Resultado esperado
- Crear rama.
- Hacer un commit claro.
- Subir cambios.
- Abrir PR con descripción simple.

## Aprendizaje
Aprender por micro-logros baja ansiedad y mejora retención.
