# Learning GitHub without panic

## Situation
"I get lost with branches, commits, and pull requests".

## Overwhelming approach
Trying to learn all of GitHub in one day with 30 concepts.

## Human-centered approach
"Teach me GitHub for one task only: push a small change and open a PR. Keep steps short and tell me what to check in each step".

## Expected output
- Create a branch.
- Make a clear commit.
- Push changes.
- Open a PR with a simple description.

## Takeaway
Learning through small wins reduces anxiety and improves retention.
