# Ideal Customer

## Purpose
Define who to serve first with clarity: segment, pains, context, language, and initial channels.

## When to use it
- When the target audience is too broad.
- When value communication is unclear.
- When early users are hard to find.

## When NOT to use it
- When the initial segment is already clear and validated.
- When the urgent issue is a critical technical blocker.

## Principles
- Focus on one specific segment first.
- Use the customer's language.
- Choose channels where the customer already is.

## Workflow
1. Define the priority segment.
2. Identify main pains.
3. Describe real usage context.
4. Adjust message and promise.
5. Select 1 to 2 initial channels.
6. Propose one outreach or test action.

## Response format
- Priority segment
- Main pain
- Real context
- Recommended message
- Initial channels
- Next action

## Boundaries
- Do not create broad generic personas.
- Do not recommend too many channels in parallel.
- Do not use jargon customers would not use.

## Quality criteria
Leaves a clear initial customer profile and one concrete action to reach them.
